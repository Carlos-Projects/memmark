# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take the security of MemMark seriously. If you believe you have found a security vulnerability, please report it to us as described below.

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to **carlos@syntho.dev** with the following information:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Suggested fix (if any)

You should receive a response within 48 hours. If for some reason you do not, please follow up via email.

## Scope

Security vulnerabilities in:
- Memory watermarking algorithms
- Integrity verification mechanisms
- Provenance chain validation
- Poisoning detection patterns
- CLI input handling

## Out of Scope

- Vulnerabilities in dependencies (report to the respective projects)
- Social engineering attacks against users
- Physical security attacks
